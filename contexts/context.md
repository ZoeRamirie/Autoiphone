# Autoiphone (AI智能外呼CRM系统)

## 项目概述
本项目旨在打造一个开源/私有的AI智能外呼与客户管理的轻量级SCRM系统，主要通过真实的本地手机号码呼出，结合大语言模型(LLM)进行智能对话、真人声音克隆(TTS)以及意向度自动打标。

## 核心业务需求
1. **通信基础**：
   - 使用用户真实的手机号码进行外呼，以保证高接通率。
   - **支持多手机号更换与轮播**：国内运营商对外呼频率有严格限制，系统必须支持多个SIM卡的轮换拨打（轮流排队、自动路由），以防止单一号码触发高频风控被封号。
2. **AI 对话管道 (AI Voice Pipeline)**：
   - 语音转文本 (ASR)：实时低延迟解析客户语音。
   - 大模型大脑 (LLM)：支持自由对话和智能销售话术。
   - 语音克隆 (TTS)：实时流式合成真人的音色进行回复。
3. **CRM 客户管理系统**：
   - **名单导入**：支持潜在客户线索（如Excel/CSV文件）的批量导入。
   - **全程录音与回溯**：完整的拨打过程将被记录保存，电话挂断后可以随时听取回放。
   - **工作流跟踪**：提供可视化的界面跟踪线索情况，支持对客户手动增加跟进记录和备忘录（Remarks）。
   - **AI打标**：电话结束后，LLM自动提取客户核心痛点、分析预算等，并打上意向度标签（例如A类、B类、C类）。

## 系统架构选型预案
- **前端 CRM**：待定（建议 Vue3 或 React + TailwindCSS）
- **后端服务**：待定（建议 Python FastAPI 或 Node.js）
- **通信引擎层 (硬件接入方案)**：
  - **方案A：多口网络 GoIP 网关**（适合高并发、设备放在机房）
  - **方案B：PC与真机协同（基于单台安卓手机，已验证拨号控制链路可行！）**
    - **设备要求**：必须使用 1 台安卓手机（不支持 iPhone），开启 USB 调试。
    - **拨号控制**：通过 USB 数据线 + `adb shell am start -a android.intent.action.CALL` 实现自动化呼出；通过 `dumpsys telecom` 精确获取电话接通/挂断状态。
    - **音频传输 (双向语音打通)**：
      - **方式一：有线直连 (极致低延迟与稳定)**。针对没有 3.5mm 耳机孔的现代手机，购买一根 **“Type-C 甚至带快充功能的音频转接线/声卡”**，再配合电脑端的 USB 独立声卡，通过两根 3.5mm 对录线实现物理打通。这是大多数中小型呼叫中心单兵桌面的首选，延迟最低，最利于 AI 执行“打断”操作。
      - **方式二：蓝牙 HFP 免提协议 (无线化，技术门槛较高)**。完全可行！将 Mac 或一台运行 Linux/Windows 的主机伪装成手机的“蓝牙耳机”。当系统探测到电话接通，电脑拦截系统层面的麦克风/扬声器流，将音频数据打包成 PCM 字节流推送给后端的 ASR/TTS。实现干净且无线的音频通道。缺点是偶尔存在由于蓝牙协商导致的 0.5 秒以内接通延迟或者偶发断连。
    - **优势**：纯真机操作，运营商极难封号。成本仅需一台旧安卓机和几十块钱的音频线缆或蓝牙配置。
- **AI 节点**：阿里云/腾讯云 ASR + DeepSeek/Qwen + GPT-SoVITS
